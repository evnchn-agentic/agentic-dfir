🥡 The Stolen Szechuan Sauce — DFIR Case 001 · Solved

2026-06-26 · DFIR Madness Case 001 (Albert Hui & Josiah Wu question set) · solved from disk + RAM only, both hosts · derived end-to-end from the artifacts — no walkthrough consulted · evidence staged & processed on homelab node .180 (~/szechuan/)

▸ New here? The 20-second story (plain language)
  • A small company (domain C137 — yes, Rick & Morty) had its secret Szechuan-sauce recipe stolen. We were handed two dead machines' disk images and RAM dumps and asked to reconstruct what happened.
  • What happened: an attacker reached the Domain Controller over the internet via Remote Desktop (RDP), dropped malware (coreupdater.exe = Meterpreter + PowerShell-Empire), gained persistence, stole the domain's password database, hopped from the DC to the Windows-10 desktop with the stolen admin password, read the secret files (the sauce, "Beth's secret"), deleted & replaced the Beth file with a decoy, and timestomped the decoy to hide it. The malware was still phoning home when the evidence was captured.
  • Every claim below traces to a specific artifact (a log entry, a registry value, a memory structure, an MFT record). Click any "evidence" toggle to see the raw proof.

③ Bonus Questions (B1–B8)

B1. CIS / SANS controls that would have prevented this? — CIS v8 #4 (secure config — no internet-facing RDP), #6 (access control / MFA), #5 (account mgmt — kill the weak passwords), #8 (centralized audit logs), #10 (malware defenses / EDR), #13 (network monitoring to catch the C2). MFA + no-public-RDP would have directly stopped the initial access.
B2. Major architecture improvement? — Remove RDP from the internet entirely; place behind VPN + MFA or an RD Gateway; segment the DC from general workstations.

B8. What file was timestomped? — C:\FileShare\Secret\Beth_Secret.txt — the decoy. Its $STANDARD_INFORMATION times were backdated ~4 h and the sub-seconds zeroed, while $FILE_NAME reveals the true creation.

The legit binary coreupdater.exe was also not stomped on either host ($SI = $FN), which is how we know the stomp was deliberate and targeted at the decoy.

▸ Methodology & toolchain (for the curious / the next analyst)
  • Scope: disk + RAM only, both hosts (Ultra-Violence tier). No PCAP / autoruns / protected-files — extracted hives, EVTX, $MFT, NTDS ourselves, as the exercise intends.
  • Integrity: all four evidence zips MD5-verified against the published hashes before processing.
  • Disk: ewfmount (E01→raw) → mmls → ntfs-3g ro mount → fls/icat/istat (Sleuth Kit); analyzeMFT for the $MFT timeline; manual $SI-vs-$FN comparison for timestomp.
  • Memory: volatility3 — windows.info / pstree / psscan / netscan / cmdline / svcscan / malfind.
  • Registry: regipy (timezone, services, Run keys, OS, decoded the Empire blob).
  • Logs: EVTX parsed via the python-evtx library (4624/4625, TerminalServices LSM 21/24, System 7045).
  • Credentials: impacket secretsdump (LOCAL: SAM + NTDS); cracked with pure-Python MD4 (impacket compute_nthash) over rockyou.
  • Discipline: every answer derived from artifacts; the published "answers"/walkthrough pages were deliberately not opened.
