⏱ CORRECTION #1 — systematic clock offset (affects every timestamp I reported)

I reported all times in host-artifact UTC and treated them as ground truth. They are true UTC + 1 hour. Subtract 1 hour from every time in the original answers to get real-world (PCAP-authoritative) UTC. Confirmed independently by both subagents and Codex.

Mechanism: the VM was set to Pacific (UTC−7/PDT) but its wall-clock held Colorado/Mountain local time; Windows therefore added 7 h instead of 6 when writing UTC into EVTX/$MFT/memory, so host artifacts read 1 h late. The PCAP router (correctly UTC−6) is the authority. The key itself shows this: the lateral RDP is "02:35:54 true / 03:35:54 host-uncorrected." My event ordering and relative timing are correct — only the absolute clock was shifted. e.g. coreupdater drop 03:24:06Z → 02:24:06 true; sauce access 03:32:21Z → 02:32:21 true.
